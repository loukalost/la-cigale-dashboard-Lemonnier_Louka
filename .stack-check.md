## 1. Stack Proposé vs. Architecture.md

### Frontend Stack

**Choix : React 18 + TypeScript + Vite**

| Critère | Validation |
|---------|-----------|
| **Framework** | React 18.2.0+ (stable, wide adoption) ✅ |
| **Language** | TypeScript 5+ (DAL type-safety, prop validation) ✅ |
| **Build Tool** | Vite (instant server start, <100ms HMR) ✅ |
| **State Mgmt** | Zustand or Context API (lightweight for CRUD focus) ✅ |
| **Component Lib** | Headless UI + Shadcn/ui components (accessibility WCAG AA) ✅ |
| **Drag & Drop** | Dnd Kit (React-native, performant Kanban) ✅ |
| **Calendar** | TanStack React Table or FullCalendar.io (Planning view) ✅ |
| **Styling** | Tailwind CSS (utility-first, responsive grid) ✅ |
| **HTTP Client** | Fetch API (native, no extra deps for CRUD) ✅ |
| **Testing** | Vitest + React Testing Library (V1, not V0) 🟡 |

**Justification :**
- ✅ Aligns with Architecture.md Section 2.3
- ✅ Best-in-class drag-drop ecosystem (react-beautiful-dnd alternative)
- ✅ Virtual list perf for 50+ reservations (react-window or TanStack)
- ✅ Dev experience: HMR < 100ms, TypeScript strict mode supported
- ✅ Hiring-friendly framework (React dominates job market)
- ✅ Bundle size: ~45KB gzip (acceptable for Render)

---

### Backend Stack

**Choix : Express.js 4.18.2 + TypeScript + Node.js 20 LTS**

| Critère | Validation |
|---------|-----------|
| **Framework** | Express 4.18.2+ (proven, minimal overhead) ✅ |
| **Language** | TypeScript 5+ (type-safe routes, DAO services) ✅ |
| **Runtime** | Node.js 20 LTS (stable, long-term support) ✅ |
| **CORS** | `cors` middleware (whitelist CORS_ORIGIN env var) ✅ |
| **Secrets** | dotenv (V0), Render env vars (prod) ✅ |
| **Airtable SDK** | @airtable/airtable NPM pkg (official SDK) ✅ |
| **Error Handling** | Custom middleware (safety sanitization) ✅ |
| **Logging** | console (dev), structured JSON (prod optional) ✅ |
| **Testing** | Jest + Supertest (V1, not V0) 🟡 |
| **API Docs** | Swagger/OpenAPI (V1, markdown for V0) 🟡 |

**Justification :**
- ✅ Aligns with Architecture.md Section 2.2
- ✅ Minimal complexity: 5 REST routes only (no GraphQL)
- ✅ DAL pattern separates Airtable layer (maintainability)
- ✅ Render native support (npm start, auto-detection)
- ✅ TypeScript DAL services = type-safe token handling
- ✅ No overkill (Fastify, NestJS would be premature optimization)

## 2. Database & External Services

| Service | Choice | Justification |
|---------|--------|---------------|
| **Primary DB** | Airtable (existing) | Source of truth, token-protected via backend ✅ |
| **Cache** | In-memory (30s TTL) | Mitigates 300 req/min rate limit ✅ |
| **Auth** | PAT (Personal Access Token) | V0 fast-track, OAuth V1+ ready ✅ |
| **Secrets Mgmt** | Render env vars | Native support, no extra tooling ✅ |
| **Deployment** | Render | Existing, familiar to team ✅ |
| **CDN/Static** | Render static routes | Vite build → Render `/dist` folder ✅ |
